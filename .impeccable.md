# The Match Signal OS — Design Context

This file is the single source of truth for all UI/UX decisions in this project. Read it before any design work. Update it when direction changes.

## Design Context

### Users

**One operator. Just David.** This is a private console, not a SaaS dashboard.

- Context: Used during matchday windows (pre-match, live, post-match) and in early-morning prep slots to assemble briefs.
- Job-to-be-done: Look at a fixture, understand the signal in under five seconds, ship Telegram + social copy in under two minutes, log accuracy after the whistle.
- The operator already knows the domain. The UI does not need to teach football, the model, or its own affordances. It needs to disappear into the work.
- Density is welcome. Hand-holding is not. Loading states, empty states, and confirmations should be terse — one line, no illustrations.

### Brand Personality

- **Three words**: editorial, composed, opinionated.
- **Voice rules already baked into product copy**: "Say what to watch, not what is guaranteed", "Use short sentences and concrete match language", "Avoid official World Cup affiliation claims", "Keep betting language out of public social copy."
- **Emotional goal**: the calm of a broadsheet matchday section the morning of a knockout tie. Authority without volume. The product should feel like it has already filed a thousand briefs.

### Aesthetic Direction

**Editorial broadsheet. FT / The Athletic / Bloomberg Businessweek with a tactical scout's discipline.** Light only — no dark mode toggle, no theme switch, commit fully to paper.

- **Reference vibe**: the print edition of the FT pink pages, The Athletic's tactical longreads, a UEFA technical report PDF, a Monocle data spread.
- **Anti-references** (do NOT do these — they are this codebase's current sins):
  - Glassmorphism — `backdrop-filter: blur` on every card. Delete.
  - Glow effects — `box-shadow: 0 0 X rgba(green, .25)` on buttons, cards, sliders. Delete.
  - Gradient text — the white-to-gray h1 gradient. Delete.
  - Neon green (#10b981) and the AI cyan-on-dark palette. Delete.
  - Cards inside cards inside cards. Flatten to rules and rhythm.
  - Dark mode default with green accents. Delete dark mode entirely.
  - Generic icon-above-heading templating. Use icons sparingly, inline, at body weight.
  - Bouncy/elastic easing on the rotating word and slider thumbs.

### Palette

Three-color discipline. Everything else is data semantics.

| Role            | Token             | Value      | Use                                                     |
| --------------- | ----------------- | ---------- | ------------------------------------------------------- |
| Paper           | `--paper`         | `#F4EFE3`  | Page canvas. Slight warm cream, never pure white.       |
| Paper raised    | `--paper-raised`  | `#FBF7EC`  | Inset panels, table backgrounds.                        |
| Ink             | `--ink`           | `#141612`  | Primary text, rules, headers. Never pure black.         |
| Ink muted       | `--ink-muted`     | `#5C5A52`  | Secondary text, labels, captions.                       |
| Ink quiet       | `--ink-quiet`     | `#8B8678`  | Hint text, disabled, helper copy.                       |
| Pitch green     | `--pitch`         | `#13513F`  | Brand accent. Eyebrows, links, key numbers. **Sparingly.** |
| Signal gold     | `--gold`          | `#B8862A`  | Single-purpose: editorial rule lines, pull-quotes, "lead story" tag. |
| Pressure red    | `--red`           | `#9F3A31`  | Data semantic only — risk, alerts, "live". Never UI chrome. |
| Analysis blue   | `--blue`          | `#254E70`  | Data semantic only — informational tags. Never UI chrome.   |

Tint neutrals toward warm ink. No grays cooler than the paper hue.

### Typography

Pair a serif display with a clean sans body. No monospace as a vibe — only for actual code/data IDs.

- **Display (serif)**: **Fraunces** (variable, optical sizing). Use the optical max for h1/h2 to get the editorial high-contrast feel. Wide tracking for eyebrows.
- **Body (sans)**: **Inter Tight** at -0.01em tracking. Replaces Plus Jakarta Sans (too rounded, too startup).
- **Mono**: **JetBrains Mono** — used only inside `<code>`, fixture IDs, and the API console. Never as decoration.
- **Drop**: Outfit (too geometric for editorial).
- **Scale** (fluid, baseline 16px / 1.5 line-height):
  - `h1`: `clamp(2.4rem, 3.6vw, 3.6rem)`, Fraunces 600, opsz 144, tracking -0.025em, line-height 1.05.
  - `h2`: `clamp(1.5rem, 2vw, 2rem)`, Fraunces 500, opsz 60, tracking -0.015em.
  - `h3`: `1.0625rem`, Inter Tight 600, tracking -0.005em, uppercase eyebrow optional.
  - `body`: `0.9375rem`, Inter Tight 400, line-height 1.6, max-width 68ch on prose, 90ch on tables.
  - `caption`: `0.75rem`, Inter Tight 500, tracking +0.04em, uppercase for labels.

### Layout & Space

- **Baseline grid**: 4px. All spacing is a multiple. Side gutters use `clamp(20px, 4vw, 56px)`.
- **Workspace max width**: 1440px, centered, with 56px gutters at desktop. Not edge-to-edge.
- **Rules over containers**: A 1px ink-muted hairline beats a card with shadow. Use `border-top: 1px solid var(--ink-muted-12)` to separate sections.
- **Asymmetry**: editorial layouts run text against a sidebar, lead with an oversized number/headline, let one column breathe while another is dense. Don't 50/50 everything.
- **Tables are first-class**: World Cup Data view is mostly tables — they get tabular-num figures, top-aligned, ink-muted thin rules between rows, no zebra striping, sticky thead with a ruled bottom.
- **Numbers belong in Fraunces**: large stat numbers (interest score, confidence, attention) use Fraunces 500 opsz 60 with tabular-nums — this is the FT cover-number move.

### Motion

- **Easing**: `cubic-bezier(0.22, 1, 0.36, 1)` (ease-out quart). No bounce, no elastic.
- **Duration**: 180ms for hover/state, 260ms for entrance, 400ms for staggered reveal. Anything longer feels slow.
- **Transform/opacity only.** Never animate height, width, padding, or box-shadow.
- **Stagger**: matchday queue rows enter at 30ms cadence on view change. Single high-impact moment, not constant micro-motion.
- Respect `prefers-reduced-motion: reduce` — drop to opacity-only at 120ms.

### Interaction

- **Buttons**: Three tiers only — primary (ink-on-paper-raised, hairline border, gold underline on hover), ghost (ink-muted, no border, gold underline on hover), destructive (red text, no fill, red underline on hover). No glow, no transform-Y on hover. The hover is the underline.
- **Inputs**: Bottom-rule only — no boxed input. `border: none; border-bottom: 1px solid var(--ink-muted-30); padding: 6px 0;` Focus draws the rule to ink full opacity, no glow.
- **Selected state**: a 2px gold rule on the left edge of the row, paper-raised background. No green glow.
- **Empty states**: one sentence in ink-muted, no illustration.
- **Loading**: a thin progress bar at the top of the workspace (1px, gold), not spinners over content.

### Design Principles

1. **Ink on paper, not pixels on glass.** No blur, no glow, no gradient backgrounds. If a card needs a shadow to look real, it's the wrong element — use a rule instead.
2. **Three colors do all the work.** Paper, ink, pitch-green. Gold appears as a hairline accent. Red and blue exist only when the data demands them.
3. **The serif carries the brand.** Numbers and headlines are set in Fraunces with tabular nums and opsz tuning. The interface gets its authority from typography, not from chrome.
4. **Rhythm > containers.** Hairline rules and vertical baseline rhythm replace card-on-card nesting. Asymmetric columns; never 50/50 by default.
5. **Density with composure.** This is a solo operator console — show everything that matters, but let each element earn its weight. No icons-above-headings, no decorative sparklines, no redundant labels.
6. **Disappear into the work.** Motion is for moments of state change, not delight. Loading is a 1px bar. Hover is an underline. The product should feel filed, not animated.
